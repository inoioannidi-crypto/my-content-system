# Skill: learn

Capture session learnings and update project memory.

## Trigger Conditions

Invoke this skill when:
- A session produced a new decision worth preserving
- A pattern came up that will likely recur
- A guideline was refined or overridden
- The user says "remember this", "note that", "let's keep track of this"
- At the end of a productive session as a wrap-up step

## Instructions

When invoked:

1. **Review the session** — Identify what was decided, discovered, or changed
2. **Categorize the learning** — Which section of `memory.md` does it belong to?
   - Key Decisions
   - Content Guidelines
   - Recurring Patterns
   - Open Questions
3. **Write concisely** — One bullet point per insight, present tense, no fluff
4. **Update `memory.md`** — Add the bullet(s) to the appropriate section
5. **Confirm** — Report back what was added and where

## What Makes a Good Memory Entry

- Specific enough to be actionable in a future session
- Explains *why*, not just *what*, when context matters
- Short — if it needs a paragraph, it belongs in a separate file with a link from memory.md

## What NOT to Capture

- Session-specific task details that won't recur
- Temporary state or in-progress notes
- Anything that duplicates existing entries

## Example

**Session insight**: "Decided to use second-person voice ('you') in all how-to content."

**Memory entry added to Content Guidelines**:
- Use second-person voice ('you') in how-to content for directness and reader engagement
